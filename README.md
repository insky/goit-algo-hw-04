# goit-algo-hw-04

## Опис проекту
Цей проект має за мету порівняння ефективності алгоритмів сортування злиттям, вставками та Timsort за часом виконання. Він реалізує кожен з цих алгоритмів сортування та вимірює час, необхідний для сортування масивів різного розміру.

## Рузультати вимірювань
Результати вимірювань часу виконання для кожного алгоритму сортування наведені в таблиці нижче:


| Розмір масиву | Сортування вставками (сек) | Сортування злиттям (сек) | Timsort (сек)  |
|---------------|----------------------------|--------------------------|----------------|
| 10            | 0.000007                   | 0.000013                 | 0.000005       |
| 100           | 0.000152                   | 0.000100                 | 0.000007       |
| 1000          | 0.014133                   | 0.001206                 | 0.000074       |
| 10000         | 1.201657                   | 0.013197                 | 0.000765       |
| 100000        | 125.392432                 | 0.160631                 | 0.009566       |


## Висновки
1. Сортування вставками є найменш ефективним серед трьох алгоритмів, особливо при збільшенні розміру масиву. Час виконання значно зростає, що робить його непридатним для великих наборів даних.
2. Сортування злиттям показує значно кращу продуктивність порівняно з сортуванням вставками, але все ж поступається Timsort.
3. Timsort є найефективнішим алгоритмом серед трьох, демонструючи найнижчий час виконання навіть при великих розмірах масивів.
4. У загальному випадку варто використовувати вбудований Timsort для сортування даних через його високу ефективність та оптимізацію для реальних сценаріїв використання.
5. Застосовувати власні алгоритми сортування є сенс тільки в специфічних випадках, коли потрібно врахувати особливості даних або вимоги до пам'яті.
